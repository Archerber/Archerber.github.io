简介

QNAP NAS 是一个网络附加存储解决方案，为个人和商业用户提供集中式的数据存储和管理。QNAP NAS 设备通常带有多个硬盘托架，可以用各种硬盘或固态硬盘组合进行填充，旨在成为一个全方位的文件共享、媒体流、数据备份和应用程序安装解决方案。

除了基本的文件共享和数据备份功能外，QNAP NAS 设备提供各种特性和功能，以满足不同的需求和喜好。这些包括能够托管虚拟机、运行 Docker 容器、作为监视系统，并同时流式传输媒体到多个设备。

此外，QNAP NAS 设备配备其自己的操作系统，名为 QTS（QNAPTurbo NAS系统），该系统具有用户友好的界面和可从 QNAP应用程序中心下载的广泛应用程序。这些应用程序涵盖从多媒体应用程序（如 Plex 和 Kodi）到生产力工具（如 Microsoft Office和Google Drive集成），甚至高性能应用程序（如虚拟化站和容器站）等各种类型。

总的来说，QNAP NAS 提供了一个强大而灵活的数据集中和管理解决方案，因此在个人和商业使用中广受欢迎。

漏洞复现

复现Qnap的漏洞时，若有存在漏洞版本而找不到相邻版本时，可以先根据QuTSCloud等的存在漏洞的相邻版本来判断，目前官方可以下载到c4.5.1.1350 - c5.0.1.2148 build 20220905之间的所有版本，通过安装QuTScloud的方式复现漏洞也不失为一种好方法，一年300r。
Qnap的许多漏洞的描述信息为了保证用户的安全一般都不会公开漏洞复现所需的关键信息，这时，需要自己进行代码审计来定位到漏洞点
思路：
1.对比两个版本中的文件的md5
2.筛选相关文件
3.对筛选出来的文件进行diff

CVE-2022-27593

漏洞信息：
据报告，影响QNAP NAS运行Photo Station的外部控制参考资源漏洞已被报告。如果利用此漏洞，攻击者可以修改系统文件。我们已经在以下版本中修复了漏洞：QTS 5.0.1：Photo Station 6.1.2及更高版本；QTS 5.0.0/4.5.x：Photo Station 6.0.22及更高版本；QTS 4.3.6：Photo Station 5.7.18及更高版本；QTS 4.3.3：Photo Station 5.4.15及更高版本；QTS 4.2.6：Photo Station 5.2.14及更高版本。

CVE-2021-44052

漏洞信息：
已经报告了一个影响运行QuTScloud、QuTS hero和QTS的QNAP设备的“链接跟踪”漏洞。如果利用，这个漏洞允许远程攻击者遍历文件系统到意外的位置并读取或覆盖意外的文件内容。我们已经在以下版本的QuTScloud、QuTS hero和QTS中修复了这个漏洞：QuTScloud c5.0.1.1998及更高版本、QuTS hero h4.5.4.1971 build 20220310及更高版本、QuTS hero h5.0.0.1986 build 20220324及更高版本、QTS 4.3.4.1976 build 20220303及更高版本、QTS 4.3.3.1945 build 20220303及更高版本、QTS 4.2.6 build 20220304及更高版本、QTS 4.3.6.1965 build 20220302及更高版本、QTS 5.0.0.1986 build 20220324及更高版本、QTS 4.5.4.1991 build 20220329及更高版本。
